    • When HTML5 has become a standard? What were the main reasons for developing new standard?
    HTML5 became a standard on October 28, 2014 as a W3C Recommendation. The main reasons for developing the new standard were to:
  • Improve support for multimedia (audio, video, canvas)
  • Enhance semantic markup
  • Provide better compatibility with mobile devices
  • Keep the language readable by humans and consistently understood by computers.

    What is a difference between W3C HTML5 and WHATWG HTML Living Standard?
  • W3C HTML5:
  • Developed by the World Wide Web Consortium (W3C).
  • Published as a recommendation.
  • Follows a more formal standardization process.
    • WHATWG HTML Living Standard:
  • Maintained by the Web Hypertext Application Technology Working Group (WHATWG).
  • Continuously updated and maintained.
  • More pragmatic and flexible approach.

    Comment on semantic and presentational markup. Why presentational markup is considered deprecated now?
  • Semantic markup refers to HTML elements that express the meaning of the content they enclose, such as header, footer, article, and section. Presentational markup was used to control the appearance of web pages with elements like font and center, but it is now deprecated because it mixes the content with its presentation. CSS is now used for layout and styling, as it separates content from design, allowing for cleaner code, better accessibility, and more flexibility in presenting the content on various devices.

    • What are the audio and video formats, officially supported by HTML standard? What happens if you use other format?
  • The audio and video formats officially supported by the HTML standard include MP3, WAV, and Ogg for audio, and MP4, WebM, and Ogg for video. If you use other formats, the browser may not be able to play the media natively, and it might require additional software or plug-ins to decode and play the content.
